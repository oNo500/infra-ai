## AI 填写指南

- **从 package.json 获取版本** — 使用 `dependencies` 和 `devDependencies` 中的实际版本，不要猜测
- **只为非显而易见的库写使用模式** — React、Express 等标准库无需说明；专用库需要
- **突出项目特有选择** — 例如"我们用 Zustand，不用 Redux"或"我们用 fetch，不用 axios"
- **记录被废弃的模式** — 若某种模式被尝试后放弃，说明原因
- **按关注点分组** — 状态、数据获取、样式、测试等
- **删除不适用的分类** — 无数据库则删除该章节
- **检查配置文件** — `tsconfig.json`、`eslint.config.*`、`vite.config.*` 揭示重要约束
