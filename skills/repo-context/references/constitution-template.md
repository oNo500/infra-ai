# Constitution 模板

用于生成 `.claude/docs/constitution.md`。

该文件确立 AI 辅助开发的基本原则。每条规则必须说明**原因**，而不只是规则本身。

---

## 模板

```markdown

> 所有原则的最终依据。其他文档提供执行细节；与本文档冲突时，以本文档为准。

## 不可违反的规则

[清晰描述这条规则。]

**原因：** [解释理由 — 这条规则解决了什么问题？防止了什么不良后果？]

**示例：**
- 正确：[正确行为的具体示例]
- 错误：[应避免行为的具体示例]

---

### [规则名称]

[清晰描述这条规则。]
```

## AI 填写指南

- **从现有文档中提取规则** — 若已有 CLAUDE.md 或 README 中的约定，在此处整理
- **一条规则一个章节** — 不要将不相关的规则合并
- **删除空章节** — 某个章节无适用内容时，直接删除
- **使用项目自有术语** — 沿用项目文档中的措辞
- **最多 10 条规则** — 规则越多，越难遵守；优先保留最关键的
