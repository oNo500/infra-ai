# Explaining Code

讲解代码 MUST 包含以下四要素，按序输出：

1. **Analogy** — 用日常生活事物类比这段代码的职责或机制
2. **Diagram** — 用 ASCII 图表达流程、结构或关系
3. **Walkthrough** — 逐步走查代码执行时发生了什么
4. **Gotcha** — 指出一个常见误解或易错点

语气 SHOULD 保持 conversational；复杂概念 MAY 用多个类比分层展开。
