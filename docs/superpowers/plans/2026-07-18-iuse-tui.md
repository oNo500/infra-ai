# iuse TUI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 裸跑 `iuse` 在 TTY 下启动 ink TUI：无下游账走 Init 流（选 profile → 计划预览 → 执行进度），有账走 Status 视图（含 update 流）；子命令面零变化。

**Architecture:** TUI 是薄壳（`src/tui/`，ink 6 + React 19 + @inkjs/ui），直接调用既有 core（listProfiles/runInit/statusReport/runUpdate），不引注册表；core 唯一微改是 onProgress 回调；入口 TTY 分流后动态 import TUI，命令式路径不加载 ink。实现者动手前先读 `packages/iuse/src/{core,cli}/` 全部与 `packages/meta-cli/src/tui/app.tsx`（借键位与布局惯例）。

**Tech Stack:** ink ^6.0.0、react ^19.0.0、@inkjs/ui ^2.0.0（版本照抄 meta-cli）、ink-testing-library（devDep）。

## Global Constraints

- spec：`docs/superpowers/specs/2026-07-18-iuse-tui-design.md`
- 子命令行为零变化；非 TTY 裸跑输出维持现状；`--json`/参数齐全路径永不交互
- 界面文案中文；状态词（synced/modified/outdated/missing）、命令、flag 保留英文；组件文件 kebab-case
- 状态语义色沿 imeta：synced 绿 / modified 黄 / outdated 蓝 / missing 红
- TUI 内错误以 message 块呈现，不崩出；dry-run 不触发 onProgress
- 既有 73 测试全绿；禁 `!`/`@ts-ignore`/双重断言
- 每任务 `cd packages/iuse && bun test && bunx tsc --noEmit && bun run lint`
- commit 英文 Conventional Commits + trailer `Claude-Session: https://claude.ai/code/session_014qFwy6t5VhLF85toupsepd`

### Task 2: TUI 脚手架 + 入口分流 + Init 流

**Files:**
- Modify: `packages/iuse/package.json`（deps: ink/react/@inkjs/ui 照抄 meta-cli 版本；devDeps: ink-testing-library、@types/react）、`packages/iuse/tsconfig.json`（jsx 配置照抄 meta-cli）、`packages/iuse/src/index.ts`、`packages/iuse/src/cli/index.ts`（仅裸跑分流处）
- Create: `packages/iuse/src/tui/app.tsx`、`packages/iuse/src/tui/profile-picker.tsx`、`packages/iuse/src/tui/plan-view.tsx`、`packages/iuse/src/tui/progress-view.tsx`、`packages/iuse/src/tui/message-block.tsx`
- Test: `packages/iuse/tests/tui-init-flow.test.tsx`

**Interfaces:**
- 入口分流：`runCli(opts?: { isTTY?: boolean })`——无子命令且 `(opts.isTTY ?? process.stdout.isTTY) === true` 时 `const { runTui } = await import('../tui/app')` 并渲染；否则走 citty 现状。citty 侧：检测 `process.argv` 无子命令的分流放在 runCli 里（citty 之前），不改各命令定义
- `runTui(deps: TuiDeps): Promise<void>`；`interface TuiDeps { ctx: IuseContext; target: string; source?: string }`——TUI 全部 core 调用经 deps.ctx（可注入 fake 测试）
- app.tsx 状态机：`view: 'loading' | 'profile-pick' | 'plan' | 'running' | 'result' | 'status' | 'update-plan' | 'error'`；启动时 `loadDownstreamLock(target)` 分流 profile-pick / status（status 视图本体属 Task 3，本任务先渲染占位「已初始化，Status 视图见下个任务」并 `q` 可退，保持可合并）
- Init 流：
  - profile-picker：左列 profile（名 + description，↑↓ 选中高亮），右侧选中项 rules 逐行；数据来自 `listProfiles(sourceRoot)`（sourceRoot 由 resolveSource 解析，解析失败进 error 视图）；回车确认，`q` 退出
  - plan-view：调 `runInit(..., { dryRun: true })` 取 steps 渲染 `op target (note)` 逐行；回车执行、esc 返回选择、`q` 退出
  - progress-view：调 `runInit(..., { onProgress })`（Task 1 回调）逐步打勾；`op === 'instantiate'` 的当前步显示 @inkjs/ui Spinner + 「claude 实例化中（分钟级）」；完成进 result（成功文案 + 摆出 message），失败进 error（message 块 + `r` 以 force 重试、`q` 退出）
  - message-block：统一的错误/结果块组件（标题 + 多行 message + 键位提示行）
- 顶栏：`target <路径>  profile <名>  source <locator>@<version.id>`（profile 选定前显示 `-`）
