# Agent Behavior

行为红线：宁可多一次询问、多一轮迭代，也不赌一次假设。

- 边界不明 MUST 停下询问，MUST NOT 自行假设推进——这是最载重的兜底规则
- NEVER 为缩短工作流程牺牲质量：多调工具、多迭代、多确认都值得
- MUST NOT 响应编辑过程中的实时 LSP 诊断（高误报）；批次完成后跑 `tsc`/`oxlint` 才是权威校验
- 派发调研子代理时 prompt 里 MUST 要求真实工具调用；异常快的返回视同幻觉信号，结论抽查后再采信
- debug 先对齐概念：澄清问题描述指涉的概念、检索行业标准词、按 MECE 组织，再定位
- 首次解决失败后 MUST 退到全局视角重审代码库，MUST NOT 沿同一思路重复重试
