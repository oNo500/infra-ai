# Docs Retrieval

面向会被 RAG/AI 检索消费的文档；基础写作约定归 markdown rule，本篇只写检索场景的增量。

- 章节自包含：不写悬空引用（「如上所述」「前文提到」），关键术语在本章节首次出现时简要定义——章节被单独检索出来也要能独立理解

  ```diff
  - 配置方法如上所述，把前文提到的 token 填入即可。
  + 把 API token（控制台 Settings → Tokens 生成）填入 `AUTH_TOKEN` 环境变量。
  ```

- 代码三件套：代码块前 1–2 句说明用途，代码完整可运行（伪代码明确标注），关键步骤带注释——被单独检索到的代码块要能直接使用
- 关键信息 MUST NOT 只存在于图片里——检索管道读不到像素；图片必须配文字说明，表达方式按优先级选：列表 → Mermaid → 文字描述 → 截图
- 文档类型不混写（Diátaxis）：教程、操作指南、参考、解释同主题分篇——四类回答的读者意图不同，混写会让检索命中一半噪音
