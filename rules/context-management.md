# Context Management

context 是工作区不是日志：陈旧工具结果与废弃岔路会拉低响应质量，要主动管理而非任其堆积。

## 子代理分派

开销大的工作交给子代理消化，只取回精简摘要，不在主上下文堆积原始结果：

- 纯调研（网页搜索、读文档）交给研究子代理
- 大范围代码库探索（扫大量文件回答一个问题）委托 Explore agent
- 多个互不相关的搜索/分析并发分派子代理
- 预期产出超过 500 行的操作一律委派

## Memory 准入

只有跨会话仍有价值的状态才进 memory：

- 存：非显而易见的已决策（「选 X 而非 Y，因为 Z」）、活跃项目上下文（截止日期、进行中的计划）、工作中发现的用户偏好
- 不存：文件内容与代码模式（读代码即可）、当前会话任务列表（用任务工具）、CLAUDE.md 或 rules 已写的内容
