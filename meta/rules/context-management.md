# 元指令：context-management rule

Claude Code 会话的上下文管理纪律，global 无条件加载。

## 目标

固化 notes 仓上下文管理笔记里的操作性规则。核心姿态一句话：
context 是工作区不是日志，陈旧工具结果与废弃岔路会拉低响应质量，
要主动管理。规则文件分层加载、CLAUDE.md 入口这类机制事实
Claude 已知，不写；只写分派判断与 memory 准入这类行为约定。

## 约束（素材，构建时组织成产物）

子代理分派（用 Agent 工具而非在主上下文堆积结果）：

- 纯调研（网页搜索、读文档）→ 研究子代理，只取回摘要
- 大范围代码库探索（扫大量文件回答一个问题）→ 委托 Explore agent
- 多个不相关的搜索/分析 → 并发子代理
- 预期产生超过 500 行输出的操作 → 委派，保护主上下文
- 模式：子代理消化开销大的工作，返回精简摘要

memory 准入（跨会话状态）：

- 存：非显而易见的已决策（「选 X 而非 Y，因为 Z」）、活跃项目上下文
  （截止日期、进行中的计划）、工作中发现的用户偏好
- 不存：文件内容与代码模式（读代码即可）、当前会话任务列表
  （用任务工具）、已写在 CLAUDE.md 或 rules 里的内容

## 产物要求

- global 落点：精简，两个主题两小节，不超过 20 行正文
- 素材源：notes 仓 `00-inbox/上下文管理.md`
