# 元指令：docs-retrieval rule

面向检索与 AI 消费的文档写作约定，作用于 `**/*.md`。
基础 markdown 写作约定归 markdown rule，本 rule 是「文档会被 RAG/AI
检索消费」场景的增量；源笔记标注过条件适用（个人笔记视是否 AI 检索
决定套用），因此独立成条按项目组合。

## 约束（素材，构建时组织成产物）

- 章节自包含：不写悬空引用（「如上所述」「前文提到」），关键术语首次
  出现时简要定义；章节被单独检索出来也能独立理解
- 代码三件套：代码块前 1-2 句说明用途，代码完整可运行（伪代码明确标注），
  关键步骤带注释
- 关键信息不能只存在于图片里：图片必须配文字说明，内容优先级
  列表 → Mermaid → 文字描述 → 截图
- 文档类型不混写（Diátaxis）：教程、操作指南、参考、解释同主题分篇，
  不在一篇里塞两种风格

## 产物要求

- scoped 落点；每条一行说清，悬空引用给一个正/反例
- 素材源：notes 仓 `20-areas/20-06-学习与成长/写作-LLM友好型文档.md`
  （可移植部分）
